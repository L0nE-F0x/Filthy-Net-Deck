## 2. Fixed this session

### 2a. Dependency vulnerability + safe updates — FIXED
`npm audit` flagged **1 critical**: vitest UI arbitrary file read/exec
(GHSA-5xrq-8626-4rwp, dev-only). Ran a semver-safe `npm update` of vitest,
react/react-dom (19.2.7→19.2.8), tailwind + @tailwindcss/vite (4.3.2→4.3.3),
@tauri-apps/plugin-store, tsx, typescript-eslint. Result: **0 vulnerabilities**,
gate still fully green. `package-lock.json` updated.

### 2b. `website/downloads/` bloat 214 MB → 46 MB — FIXED
The repo was carrying every historical installer (9 dmgs + 9 exe+sig sets).
Mirrored all 8 superseded versions (2.0.1–2.4.1) to their **GitHub Releases** as
assets (also re-uploaded current 2.4.2 + 2.5.0 for consistency), then
`git rm`'d the superseded files. Only current + previous (2.4.2, 2.5.0) remain in
the tree. Old-version download links on the marketing site were already pointing
at GitHub, so nothing user-facing breaks. **Note:** this only shrinks the tree
going forward — the 1.1 GB `.git` history still holds every past binary blob (see
`docs/GIT-HISTORY-BLOAT.md`; a filter-repo purge is the owner's call, deferred).

### 2c. meta-web deck pages linked version-pinned binaries — FIXED
`pipeline/build-meta-site.mjs` baked `../downloads/Filthy-Net-Deck-Setup-<ver>.exe`
into all 35 static deck/hub pages. Those pages are **never regenerated on release**
(only by the meta cron), so the links rot and 404 the moment an installer is pruned
(exactly what 2b would have caused — the pages still pointed at 2.2.1). Changed the
download links to `index.html#download`, which is always current. Regenerated all
35 pages.

### 2d. Daily cron never rebuilt the public meta-web pages — FIXED (freshness bug)
`daily-meta.yml` ran `npm run meta` + `npm run sets` and **staged**
`website/meta-web/` in the commit, but never ran `npm run meta:site` — so the
static public deck pages silently drifted a day behind the live feed every day.
Added a "Rebuild public meta-web pages" step. This is the freshness directive's
exact failure mode (`docs/MAINTENANCE.md`), now closed.

### 2e. 8 dead CSS classes removed — FIXED
Scanned all 772 CSS classes against every `.ts/.tsx` + `index.html`. Removed the
fully-unreferenced set: the old iframe-viewer family (`.viewer-overlay/-panel/
-header/-url/-frame/-hint`, from a removed in-app browser), legacy `.star-btn`
(favorites star, superseded), and `.skeleton-banner`. Bundle CSS dropped ~2 kB.
Everything else is either referenced or a documented dynamic-prefix class.

## 3. Backlog — for the successor models (prioritized)

### P1 — worth doing soon

- **`src/pages/Stats.tsx` is 1889 lines.** Biggest file in the app by far and the
  one most likely to cause merge pain / model context blowups. It holds ~8 sub-
  components (`SummaryTiles`, `FormTiles`, `SplitsPanel`, `StatsArsenal`,
  `DeckBreakdown`, `MatchHistory`, `DeckDetail`, `StatusPanel`). Extract each into
  `src/components/stats/` with the shared helpers (`tally`, `RateBar`,
  `winrateFavor`) in one module. Pure mechanical refactor, no behavior change —
  ideal for a cheaper model. Guard with the existing `statsHelpers.test.ts` +
  a browser render check via `window.__fndStore`.
- **Major dependency bumps deferred** (need real testing, do one at a time on a
  branch): `typescript` 5.8→7.0, `vite` 7→8, `@vitejs/plugin-react` 4→6,
  `vitest` 3→4. None are security-driven; all are behind semver majors. Do NOT
  batch — each can break the build/test harness independently.

### P2 — nice to have

- **No Rust coverage for `arena.rs` beyond the stem-match unit test.** The 4s
  process-poll (`MTGA` exact-stem match) has one test; the `arena:running`
  emit path and debounce are untested. Low risk (owner-verified live) but a gap.
- **`docs/MAINTENANCE.md`** still reads as if meta-web rebuilds were already
  automatic. After 2d that's now true, but the doc predates the bug — give it a
  once-over so the "fully automatic" table explicitly lists meta-web.
- **Secondary-monitor limitation** (`toast` + `presence` webviews anchor to the
  primary monitor; Arena on a second display puts them on the wrong screen).
  Owner-aware, unstarted. Real work (follow Arena's window across monitors),
  correctly deferred — flagging so it isn't rediscovered as a "bug."

### P3 — owner-scoped, do not start without asking

- **Donations link** (Ko-fi / Buy-Me-A-Coffee, no backend needed) — repeatedly
  noted as unstarted across handoffs.
- **v3.0 accounts + cross-device sync** — the intended next-major anchor.
- **Scryfall attribution** was removed from Settings in v2.2.1 (their API
  guidelines *request* attribution). This was a deliberate owner trade-off, not
  an oversight — left as-is, but re-flagging so a future model doesn't "helpfully"
  re-add or permanently bless it without the owner.
- **`.git` history is 1.1 GB** from historical binary blobs (see 2b + existing
  `docs/GIT-HISTORY-BLOAT.md`). A `git filter-repo` purge would reclaim it but
  rewrites history / breaks clones — owner's call.

## 4. Architecture notes worth carrying forward

- **Four webviews:** `main` · `overlay` (match HUD) · `toast` (fullscreen-proof
  alert card) · `presence` (corner badge). **Every new window label MUST be added
  to `src-tauri/capabilities/default.json` `windows`** or it gets zero core
  permissions. `.transparent()` stays `#[cfg(not(macos))]` or the dmg CI breaks.
- **Preview/verification recipe** (no Arena needed): `npm run dev`, then in the
  browser console seed `window.__fndStore.setState({ trackerMatches: [...] })`
  with real Arena grpIds (e.g. Ethereal Armor 92065). The overlay/toast/presence
  windows have a demo route: `/?demo#/overlay`. Browser-pane **screenshots work**
  now (they timed out in older sessions on Scryfall CDN images) but DOM /
  `get_page_text` is still the reliable check.
- **Single feed entry point:** `normalizeMetaBundle` in
  `src/services/deckHelpers.ts` — source sanitization, listNote stripping, and
  all feed normalization happen there. Don't scatter feed logic elsewhere.
- **Release train is mandatory and fully documented** in the `release-workflow`
  memory. Sign only with the key whose id is `67FCA9900F523D49`; the repo-root
  `filthy-net-deck.key` is an abandoned key that produces valid-looking sigs that
  break auto-update.
